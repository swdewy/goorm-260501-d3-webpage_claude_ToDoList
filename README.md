# IDO List

> "기능이 많아서 삭제했던 앱들을 보며, 뺄 수 있는 건 다 뺀 To Do 앱"

---

## 삭제의 철학

이 앱의 정체성은 **추가한 것**이 아니라 **뺀 것**에 있다.

시중의 To Do 앱은 기능이 많아질수록 사용자의 집중을 방해하는 역설에 빠진다.  
포모도로, 협업, AI 추천, 무한 카테고리 — 이 모든 것이 결국 앱을 **삭제하게 만드는 원인**이 된다.

이 프로젝트는 그 반대 방향을 선택했다.  
입력 1초, 완료 1탭. 생각이 행동이 되는 마찰 제로의 경험.

---

## 뺀 것들, 그리고 그 이유

기능을 빼는 것은 능력이 없어서가 아니라, 의도적인 결정이다.

| 제거한 기능 | 제거 이유 |
|-------------|-----------|
| 알림 · 리마인더 | 앱이 사용자를 조종하지 않는다. 할일의 우선순위는 사람이 결정한다. |
| 포모도로 타이머 | 집중은 앱이 만드는 것이 아니라 사람이 만드는 것이다. |
| AI 자동 추천 | 무엇을 할지는 내가 결정한다. AI는 보조 수단이지 주체가 아니다. |
| 팀 협업 · 공유 | 이 앱은 나만의 공간이다. 타인의 시선은 집중을 방해한다. |
| 무한 카테고리 | 분류가 할일보다 많아지는 순간 앱은 실패한다. 최대 6개로 강제 제한. |
| 외부 캘린더 연동 | 복잡성을 줄이는 것이 목표다. 단일 진실 공급원(LocalStorage)만 유지한다. |
| 계정 · 로그인 | 로컬 우선 철학. 사용자 데이터는 사용자 브라우저 안에 있다. |

## 남긴 것들

### 핵심 기능

- **할일 CRUD** — 추가 · 완료 · 삭제 · 더블클릭 인라인 수정
- **3가지 상태** — 예정 → 진행중 → 완료 (원 클릭 순환)
- **카테고리 필터** — 커리어 · 자기계발 · 집안일 · 재무 (최대 6개 강제 한도)
- **오늘 뷰 / 전체 뷰** — 하단 내비게이션으로 전환
- **LocalStorage 영속성** — 서버 없이 브라우저에 데이터 보존
- **다크모드** — OS 자동 감지 + 수동 토글

### D3.js 시각화 (인사이트 탭)

경쟁앱 5종(Todoist · TickTick · Things 3 · MS To Do · Notion) 모두  
진행률 바 수준에서 멈춘다. 이 앱은 D3.js로 세 가지 차트를 제공한다.

| 차트 | 설명 |
|------|------|
| 완료 히트맵 | 26주 날짜별 완료 건수 — GitHub 잔디 스타일 |
| 주간 추세 | 최근 8주 완료 건수 막대 차트 |
| 카테고리 도넛 | 카테고리별 완료 비율 인터랙티브 차트 |

## 기술 스택

```
HTML5       — 시맨틱 마크업, WCAG AA 접근성
CSS3        — CSS 변수(Custom Properties) 테마 시스템
Vanilla JS  — ES2022+, 프레임워크 없음
D3.js v7    — 데이터 시각화 전용 (UI 조작에 사용 금지)
LocalStorage — 유일한 데이터 저장소
GitHub Pages — 빌드 도구 없는 정적 배포
Pretendard  — 한글 최적화 폰트 (CDN)
```

프레임워크 없음. 빌드 도구 없음. 브라우저 표준 API만으로 구현한다.  
바이브코딩 입문자가 코드를 읽고 학습할 수 있는 수준을 의도적으로 유지했다.

## 데이터 구조

```js
// 할일 객체
{
  id:          "uuid-v4",          // crypto.randomUUID()
  text:        "할일 텍스트",
  status:      "todo",             // "todo" | "inprogress" | "done"
  category:    "career",           // "career" | "dev" | "home" | "finance" | null
  createdAt:   "2025-05-01T09:00:00.000Z",
  completedAt: null,               // 완료 시 ISO 8601
  order:       0
}
```

```
LocalStorage 키
d3todo_tasks     → Task[]
d3todo_settings  → { theme, defaultView, customCategories }
d3todo_version   → "1.0"
```

## 로컬 실행

```bash
# Python 3
python3 -m http.server 4567

# 브라우저에서 열기
open http://localhost:4567
```

빌드 과정 없음. `index.html`을 브라우저에서 직접 열어도 동작한다.

## 디자인 원칙

| 원칙 | 적용 |
|------|------|
| 60% 여백 룰 | 화면 면적의 60% 이상을 Negative Space로 유지 |
| 카드 시스템 | 모든 정보는 흰색 카드(radius: 20px) 위에 배치 |
| 단일 강조색 | 포레스트 그린 `#1A7A4A` 하나로 모든 강조 처리 |
| WCAG AA | 모든 텍스트-배경 조합 대비율 4.5:1 이상 |
| 인터랙션 3상태 | Hover · Active · Focus-visible 필수 정의 |

## 카테고리 시스템

```
기본 4개: 커리어 | 자기계발 | 집안일 | 재무
커스텀:   최대 2개 추가 가능
총 한도:  6개 초과 시 앱이 코드 레벨에서 추가를 거부
```

최대 6개 한도는 "분류가 할일보다 많아지면 실패한다"는 철학을  
코드로 강제한 것이다. 의도적인 제약이다.

## GitHub Pages 배포

1. 저장소를 GitHub에 `public`으로 생성
2. **Settings → Pages → Source**: `main` 브랜치, `/ (root)` 선택
3. Push 후 `https://[username].github.io/[repo-name]/` 접근

배포에 서버, 빌드 도구, 패키지 매니저가 필요하지 않다.
